HTML+CSS+JavaScript

HTML

HTML简介

  1. 什么是HTML?

    HTML 指的是超文本标记语言(HyperText Markup Language),是用来描述网页的一种语言。

  2. 大多数HTML元素包含开标签和闭标签，两者之间又包含纯文本或其他HTML标签。
  3. HTML元素能被嵌入其他标签，被包围的元素是外围父元素的孩子。
  4. 任何可见内容必须放在<body>标签的开闭标签之间。
  5. 标题<h1>到<h6>。
  6. <p>(paragraph)，<span>与<div>(division)标签表示文本或块元素。
  7. <em>与<strong>用来强调文本。
  8. 换行用<br>(line break)标签。
  9. <ol>(ordered lists)是有序表，<ul>(unordered lists)是无序表。
  10. <img>(images)与<video>(videos)分别连接图片与视频资源。

HTML文档标准

  1. <!DOCTYPE html>需在html文件首行，告诉浏览器HTML的版本。
  2. <html>元素包含所有html代码。
  3. 网页的信息，如标题等，定义在<head>中。
  4. 网页的标题定义在head中的<title>里。
  5. 网页的标题显示在浏览器的标签栏里。
  6. <a>(anchor)标签用来连接外部网页，内部网页或者当前网页其他部分内容。
  7. 可以通过<a>结合id跳转到当前网页的其他部分。
  8. 通过空格和缩进增强可读性。
  9. 用<!-- comment -->注释。
  10. 更多html标签参考Mozilla documentation。

HTML表格

  1. 用<table>标签制作表格。
  2. <tr>(table rows)添加表格行。
  3. <td>(table data)添加表格数据。
  4. <th>(table heading)添加行标题或列标题。
  5. colspan属性设置跨列单元格。
  6. rowspan属性设置跨行单元格。
  7. 表格可分为三部分：头，主体，脚。
  8. 表头用<thead>表示。(表格标题)
  9. 主体用<tbody>表示。(表头与数据)
  10. 脚用<tfoot>表示。(表格描述)

HTML表单

  1. <form>表示表单。
  2. <form>的action属性设置表单提交到哪。
  3. <form>的method属性设置表单提交方式。
  4. <input>元素：
    • type属性为"text"是单行文本输入框。
    • type属性为"password"是密码输入框。
    • type属性为"number"是数值输入框。
    • type属性为"range"是滑动条。
    • type属性为"checkbox"是复选框。
    • type属性为"radio"是单选框。
    • type属性为"list"搭配<datalist>可输入的下拉框。
    • type属性为"submit"是提交表单按钮。
  5. <select>是不可输入的下拉框，<option>设置选项。
  6. <datalist>是配合<input>使用的数据表，<option>设置选项。
  7. <textarea>是大的文本输入框。
  8. 当表单被提交时，各个元素的name与value会以name=value的形式成对发送。
  9. placeholder属性可以设置提示信息，当用户键入或选中时消失。

HTML表单客户端校验

  1. 客户端校验发生在表单被送到服务端之前。
  2. requried属性要求必须有输入，不可为空。
  3. min属性规定输入数值的最小值。
  4. max属性规定输入数值的最大值。
  5. maxlength属性规定文本输入长度的最大值。
  6. minlength属性规定文本输入长度的最小值。
  7. pattern属性要求输入匹配相应的正则式。
  8. 如果表单校验不通过，用户会得到相应的提示信息。

含语义的HTML

  1. 含语义的HTML通过特定标签内包含特定内容的方式告诉页面其含义。
  2. 含语义的HTML是一种标准，使网页便于人们阅读，改善SEO。
  3. <header>、<nav>、<main>、footer构成基本的网页结构。
  4. <section>定义文档同一主题下的元素，例如章节、标题、或文档的其他区域。
  5. <article>包含文章、博客、评论等内容。
  6. <aside>包含与主要内容相关，但对理解主要内容来说又不是必须的。
  7. <figure>用来包装各种类型的媒体资源。
  8. <figcaption>用来描述<figure>的标题。
  9. <video>、<embed>、<audio>用来打开媒体资源。

CSS

CSS的建立与选择器

  1. CSS(cascading style sheet)即层叠样式表能改变HTML元素的样式，CSS必须先选择HTML元素，然后将样式应用到所选元素上。
  2. CSS可通过标签名，类名，ID名选择HTML元素。
  3. 一个HTML元素可以有多个类名，用空格隔开。
  4. 类可以多次使用，ID只能有一个。
  5. 优先级：ID选择器>类选择器>元素选择器。
  6. 多个选择器可串联起来选择元素，这会提高它的优先级。(如div.classname，div#IDname)
  7. 内嵌元素可以通过后代选择器选择，即选择器间用空格隔开。
  8. 属性值加上!important后优先级最高，几乎很难被覆盖。
  9. 多个选择器可共用同一样式代码，选择器间通过逗号隔开。

CSS规则

  1. CSS声明由属性与值组成。
  2. font-family属性定义元素的字体。
  3. font-size控制文本的字体大小。
  4. font-weight定义文本字体粗细，通常有bold与normal两个值。
  5. text-align属性定义文本对齐方式，通常有left，right，center三种方式。
  6. 文本有两种不同的颜色属性color与background-color，color定义文本前景色，background-color定义文本背景色。
  7. CSS通过opacity属性控制元素透明度，为1时是可见的、不透明，为0时是不可见的、透明的。
  8. CSS通过background-image属性设置元素背景图片。

CSS盒模型

  1. 盒模型由内容，内边距，边框，外边距组成，如下图。
  2. 内容的宽和高以px或百分比为单位。
  3. 边框内包含内边距与内容区域。边框的粗细、样式、颜色可以通过CSS属性设置。
  4. 填充区(内边距)是内容与边框之间的区域，以px或百分比为单位。
  5. 外边距是边框外的区域。

  7. 垂直方向的外边距坍塌(合并)，总距离等于相邻元素中比较大的外边距，如下图。
  8. margin: 0 auto使元素在父元素中水平居中，如果它有宽度的话。
  9. overflow属性可被设置成display、hide、scroll,规定该元素超出父元素后如何渲染。
  10. visibility属性可以隐藏或显示元素。

    visibility:hide与display:none不同于：visibility:hide元素只是不可见，所占空间还在，display:none元素不仅不可见，所占空间也被删除。

CSS展示与定位

  1. position属性允许通过三种方式定位元素，默认为static。
  2. position设为relative时，元素相对默认位置进行偏移，还在HTML文档流中。
    • top - 元素顶部偏移距离，即元素向下移动距离。
    • bottom - 元素底部偏移距离，即元素向上移动距离。
    • left- 元素左部偏移距离，即元素向右移动距离
    • right - 元素右部偏移距离，即元素向左移动距离
  3. position设为absolute时，元素相对最近的父元素进行偏移，脱离HTML正常文档流。当页面滚动时，该元素会随之滚动。
  4. position设为fixed时，元素会固定在页面的某一位置，不随页面滚动。
  5. z-index规定元素的层级，层级高的元素会覆盖其它层级低的元素，对static的元素不起作用。
  6. display属性设置元素为内联元素(inline)、块级元素(block)、内联块(inline-block)。
  7. inline元素占尽量小的空间，无法设置width与height属性。
  8. block元素占满容器的宽度，可以设置height属性。
  9. inline-block可以设置width与height属性，它们挤在一起，不占满容器宽度。
  10. float属性可使元素在页面的最左边或最右边。
  11. 可以用clear属性清除元素左边或右边或两边的浮动。
    • left – 元素左边不允许接触容器中的其他元素。
    • right– 元素右边不允许接触容器中的其他元素。
    • both – 元素两边不允许接触容器中的其他元素。
    • none – 元素两边允许其他元素。

CSS颜色

在CSS中一共四种设置颜色的方法：

  1. 颜色名–一共147种颜色名。
  2. 16进制表示颜色：
    • 十六进制为09，剩下1015用A~F表示。
    • 16进制颜色以#开头，接上红绿蓝颜色值，如#23F41A。
  3. RGB(red,green,blue)
    • RGB颜色用rgb()表示，三个参数依次表示红、绿、蓝的颜色值。
    • RGB的范围从0到255，例如rgb(7, 210, 50)。
  4. HSL(hue,saturation,lightness)
    • HSL分别表示色彩、饱和度、亮度。
    • 色彩范围从0到360，饱和度与亮度用百分比表示，例如hsl(200, 20%, 50%)。
  5. 可以在RGB与HSL中加上第四个值，a(Alpha)透明度，范围从0到1。如rgba(7, 210, 50, 0.5)或hsla(200, 20%, 50%, 0.5)

CSS字体与排版

  1. 字体粗细用font-weight设置，范围从100到900，值为100的倍数。
    • 400是默认值normal。
    • 700表示值bold。
    • 300表示值light。
  2. font-style值为italic时字体倾斜。
  3. 行间距可用行高line-height设置。
  4. 有衬线字体(serif)字末端有细节，而无衬线字体(sans-serif)没有。
  5. 当用户计算机没有安装所选字体时，选用备用字体(Fallback fonts)。
  6. 可用<link>在HTML中引入字体库，或用@font-face在CSS中引入字体库。
  7. word-spacing用来设置词间距。
  8. letter-spacing用来设置字间距。
  9. text-align用来设置文本对齐方式。

CSS网格

  1. 容器元素需设置display为grid;
  2. grid-template-columns定义网格的列数与列宽，声明时只列出列宽，有几个列宽表示有几列，特定单位fr表示可用空间的片段，若有n个片段，每个片段占1/n的长度。
  3. grid-template-rows定义网格的行数与行高，声明时只列出行高，有几个行高表示有几行。
  4. grid-template将grid-template-rows与grid-template-columns合并定义，用/隔开。
  5. grid-gap设置行间隙与列间隙。
  6. grid-row-start与grid-row-end设置元素横跨那些行，若横跨第4、5行，截至行为6。
  7. grid-column-start与grid-column-end设置元素横跨那些列。
  8. grid-area将grid-row-start、grid-column-start、grid-row-end与grid-column-end按顺序合并，用/隔开。

CSS网格进阶

  1. grid-template-area可以设置网格区域及其名称。声明一个名称为一个区域，同一行的区域在同一双引号中。
  2. justify-items设置元素在网格中水平方向的对齐方式，在容器中设置。
    • start–元素在网格中左对齐。
    • end–元素在网格中右对齐。
    • center–元素在网格中水平居中。
    • stretch–元素水平拉伸充满网格，如果某一元素有特定的宽将不做拉伸。
  3. justify-content设置网格在容器中水平方向的对齐方式，在容器中设置。
    • start–网格在容器中左对齐。
    • end–网格在容器中右对齐。
    • center–网格在容器中水平居中。
    • stretch–网格水平拉伸充满容器。
    • space-around–水平方向上，用空白填充网格两边，会使网格间空白是边缘空白的两倍。
    • space-between–水平方向上，用空白填充网格间隙，边缘没有空白。
    • space-evenly–水平方向上，用空白均等地填充网格两边与边缘。
  4. justify-self规定某一个元素的水平对齐方式，在元素中设置，值同justify-items。
  5. align-items设置元素在网格中垂直方向的对齐方式，在容器中设置，值与justify-items类似。
  6. align-content设置网格在容器中垂直方向的对齐方式，在容器中设置，值与justify-content类似。
  7. align-self规定某一个元素的垂直对齐方式，在元素中设置，值同align-items。
  8. grid-auto-rows设置自动添加的行的行高，增加网格可变性。
  9. grid-auto-columns设置自动添加的列的列宽，增加网格可变性。
  10. grid-auto-flow设置元素个数超出网格数时，网格自动添加的方式。
    • row–自动添加行。(默认)
    • column–自动添加列。
    • dense(密集的)–如果有小块元素会通过算法填充到之前留下的洞中，可以与row、column搭配使用，例如grid-auto-flow: row dense;。

对象

  1. 对象是键值对的集合，每个键值对都是一个属性，当属性为函数时被称作方法。

  8. 可通过For...in遍历对象。

对象进阶

  1. 方法所属的对象称为调用对象，this指调用对象，可被用来取得调用对象的属性。

  2. 方法不会自动取得调用对象的其他内部属性。

  3. this不能用在箭头函数中，因为箭头函数中的this指全局对象，而不是调用对象。

  4. JavaScript对象没有内嵌的私有属性，可在属性名前加_表示私有属性，告诉其他开发者不能直接改变该属性。即使如此，它还是可以被直接改变。

Javascript Promise(承诺将来会执行)

  1. Promise是Javascript对象，代表一个异步操作的最终结果。
  2. Promise有三种状态：pending(待定)、resolved(解决)、rejected(拒绝)。
  3. 当Promise为resolved或rejected时是确定的(settled)。
  4. 我们通过关键字new构建Promise对象，并向构建方法传递执行函数作为参数。
  5. setTimeout()是一个Node函数，用事件循环(the event-loop)延迟回调函数的执行。
  6. .then()以成功处理时的回调函数为参数，包含Promise解决后所做的事。
  7. .catch()以失败处理时的回调函数为参数，包含Promise拒绝后所做的事。
  8. 可以将.then()与.catch()串联起来，用于处理有依赖关系的按顺序的多个任务。
  9. 为保证Promise组合的正确，每个.then()中都要返回一个Promise。
  10. 串联Promise比内嵌Promise好。
  11. 同时执行多个异步操作用Promise.all()，用于处理并行的多个任务。

ASYNC AWAIT(异步等待)

  1. async...await是由Javascript promise生成的语法糖(通常来说使用语法糖能够增加程序的可读性，从而减少程序代码出错的机会)。
  2. 我们用关键字async声明异步函数。
  3. 在async函数中我们用await暂停当前函数直到异步操作完成。
  4. await返回Promise中的resolved value(解决值)。
  5. 我们可以像同步语法一样用多个await。
  6. 在async函数中可用try...catch语句处理错误。
  7. 在async函数中可用await Promise.all()处理并行异步操作。

Request I

  1. JavaScript是一种具有异步能力的网络语言。AJAX表示Asynchronous JavaScript and XML，是一种用来实现Javascript异步能力的工具。
  2. HTTP具有许多种请求方式，主要有GET与POST。
  3. GET方法只向其他来源请求信息。
  4. POST方法除了只向其他来源请求信息，还向其他来源传输信息。
  5. GET请求能用XMLHttpRequest对象与Javascript来写。
  6. POST请求也能用XMLHttpRequest对象与Javascript来写。
  7. GET与POST请求需要用new创建一个XHR对象，设置其responseType，创建一个函数处理响应对象(response)，以及打开(open)与发送(send)请求。
  8. 为了在URL末端添加查询字符串(query string)可用?再添上参数。
  9. 为了提供多个参数，用&隔开键值对，用=赋值。

Request II

  1. GET与POST请求有许多种创建方式。
  2. fetch()与async/await是用来异步请求数据的函数。
    • fetch+GET
    • fetch+POST
    • async+await+GET
    • async+await+POST
  3. fetch()是个web API，用来创建request，fetch()返回一个Promise。
  4. 我们可以串联.then()来处理fetch()返回的Promise。
  5. .json()方法将Promise转换为JSON对象。
  6. async用来创建返回Promise的函数。
  7. await用来告诉程序当Promise解决时再继续。
  8. await只在async函数中使用。
